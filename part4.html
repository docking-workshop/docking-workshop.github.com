Step 11 - Analyzing the set2_dock1

  • [MOE] → File → Open → Open → set2_dock1.mdb → OK

    • This database contains the 11 199 docked compounds without the activity. In order to retrieve this field and therefor compute the enrichment we need to merge this database with another one containing the activity field for those compounds.

    A - Merging

  • [DBV] → Merge...

    • Input Database 1: set2_dock1.mdb
    Input Database 2: set2_activity.mdb

  • ... → Next >>

  • ... → Next >>

  • ... → Merge

    • n.b. Now you can see a new column "Activity" in the database.

    B - Sorting

    Now let's sort the database according the Score.
  • [DBV] → right-click on "S" → Sort → Ascending → Yes
    • We will now save the Activity field in a separated file, so we can compute the enrichment of this run.
  • [DBV] → left-click on "Activity"
  • [DBV] → File → Save as...

    • Output: set2_dock1_ranked.list

  • ... → Ok

    • C - Computing enrichment

    Now visit our tool in order to compute enrichment.
  • Choose file → set2_dock1_ranked.list → Submit

    • n.b. The value of the enrichment can be found in front of "Calculated AUC is". You can also download a .pdf that contains the enrichment curve.

    Step 12 - Analyzing the set2_dock2

    Repeat the same procedure with set2_dock1.mdb.
  • Merge the Activity field from set2_activity.mdb into set2_dock1.mdb.
  • Once sorted, export the Activity field into a new file set2_dock1_ranked.list.
  • Compute the enrichment
