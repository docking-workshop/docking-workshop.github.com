Step 8 - Standard docking

After creating a receptor and a set of ligands to dock to that receptor, we have to set up the actual docking run. Docking algorithms usually allow to set a large number of parameters that will steer the docking process. Here, we're going to start with a very simple docking example to explain the basics of docking. Docking in MOE is controlled via
  • [MOE] → Compute → Dock...

    • Ligand: set1_washed.mdb
    Output: set1_docked.mdb
  • ... → Run

    • The docking will again result in a database which is MOE's way of handling a number of molecules. This database will contain the docked poses of the molecules, the \textit{mseq} and a number of scores from the docking run. The mseq tells you to which molecule in the original database the respective pose belongs. This is necessary in case you saved multiple poses per ligand.
    The docking poses can be browsed in [MOE] conveniently using the database browser.
  • [DBV] → File → Browse...

    • The interactions of the molecule with the receptor can be visualized in 2D via
  • [MOE] [RHS] → Ligand → Ligand interactions... → select "Browser Atoms" → Apply

    • Step 9 - Rescoring

    The scoring functions used in a docking process are designed to be quite fast. The docking tool can be used to rescore the function using a more sophisticated scoring function. At the same time we'll limit our database to one pose per ligand.
  • [MOE] → Compute → Dock...

    • Ligand: set1_docked.mdb
    Output: set1_rescored.mdb
  • ... → Run

    • Step 10 - Refinement

    In a second docking run, we will try to address some problem arrising with the aforementioned docking. This second docking will use caffeine reference for the binding pocket and we'll will use the forcefield implementations in MOE to improve docked poses (Refinement) and use a even more sophisticated scoring function than before (Rescoring).
  • [MOE] → Compute → Dock...

    • Ligand: set1.mdb
    Output: set1_refined.mdb
  • ... → Run
